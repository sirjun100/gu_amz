# Telegram API 申请机器人 2.0 需求文档

## 1. 项目概述

### 1.1 项目背景
当前 Telegram API 申请流程较为繁琐，用户需要手动操作多个步骤才能完成申请。本项目旨在开发一个功能完善的 Telegram 机器人，简化 API 申请流程，同时提供充值、卡密兑换等增值服务，为用户提供一站式的 API 申请解决方案。

### 1.2 项目目标
- 提供便捷的 Telegram API 申请服务
- 支持多语言（中文/英语）切换
- 实现余额管理、充值、卡密兑换等功能
- 提供友好的用户界面和流畅的操作体验
- 确保系统稳定性和安全性

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 多语言支持
- 支持中文和英语双语言切换
- 根据用户首次使用的语言自动设置默认语言
- 提供语言切换选项

#### 2.1.2 主菜单
- 点击 /start 命令后显示主菜单
- 主菜单包含以下按钮：
  - 申请 Telegram API
  - 个人中心
  - 充值余额
  - 卡密兑换
  - 帮助说明
  - 联系客服
- 显示当前余额和申请单价信息

#### 2.1.3 API 申请流程
- 选择申请数量（1个或5个起）
- 余额充足时直接扣除费用并进入申请流程
- 余额不足时提示充值
- 申请流程：
  1. 发送手机号（含国际区号）
  2. 接收 Telegram 验证码
  3. 输入验证码
  4. 系统自动完成申请并返回 api_id 和 api_hash

#### 2.1.4 充值功能
- 支持自定义充值金额
- 提供充值优惠（充值20u以上赠送10%）
- 支持多种支付方式：
  - 支付宝扫码
  - USDT转账（TRC20）
  - TRX转账
- 显示充值二维码和收款地址
- 自动监听到账情况

#### 2.1.5 卡密兑换
- 支持输入卡密兑换余额
- 卡密类型包括1u、5u、10u等
- 兑换成功后自动更新余额

#### 2.1.6 个人中心
- 显示用户 TG ID、用户名
- 显示当前余额、累计充值、累计申请次数
- 提供返回菜单按钮

#### 2.1.7 帮助说明
- 提供详细的使用说明
- 包含注意事项
- 提供返回菜单按钮

#### 2.1.8 联系客服
- 直接打开客服的 Telegram 账号

### 2.2 辅助功能

#### 2.2.1 余额管理
- 实时显示余额
- 记录充值和消费历史

#### 2.2.2 订单管理
- 生成充值订单
- 跟踪订单状态
- 订单24小时有效

#### 2.2.3 系统通知
- 充值到账通知
- API 申请成功通知
- 余额变动通知

## 3. 非功能需求

### 3.1 性能要求
- 响应时间：用户操作后1秒内响应
- 并发处理：支持1000+用户同时在线
- 系统稳定性：99.9%的可用性

### 3.2 安全要求
- 数据加密：用户信息和交易数据加密存储
- 支付安全：确保支付过程安全可靠
- 防作弊：防止恶意刷取 API 和充值

### 3.3 兼容性要求
- 支持各种 Telegram 客户端（手机、桌面端）
- 支持不同网络环境下的使用

### 3.4 可扩展性
- 易于添加新功能和支付方式
- 支持未来业务扩展

## 4. 流程说明

### 4.1 主流程
1. 用户发送 /start 命令
2. 机器人显示主菜单（多语言支持）
3. 用户选择功能：
   - 申请 Telegram API
   - 个人中心
   - 充值余额
   - 卡密兑换
   - 帮助说明
   - 联系客服

### 4.2 API 申请流程
1. 用户点击「申请 Telegram API」按钮
2. 进入选择数量页面
3. 用户选择申请数量（1个或5个起）
4. 系统检查余额：
   - 余额充足：扣除费用，进入申请页面
   - 余额不足：进入余额不足界面
5. 申请页面：
   - 用户发送手机号
   - 系统发送验证码
   - 用户输入验证码
   - 系统完成申请并返回 API 信息

### 4.3 充值流程
1. 用户点击「充值余额」按钮或因余额不足进入充值页面
2. 进入充值页面
3. 用户点击「输入充值金额」按钮
4. 进入输入金额页面，用户输入金额
5. 进入选择支付方式页面，用户选择支付方式
6. 进入对应支付页面，显示支付信息
7. 用户完成支付
8. 系统确认到账并更新余额
9. 返回主菜单

### 4.4 卡密兑换流程
1. 用户点击「卡密兑换」按钮
2. 进入卡密兑换页面
3. 用户发送卡密
4. 系统验证卡密并兑换余额
5. 显示兑换结果
6. 返回主菜单

## 6. 技术要求

### 6.1 技术栈
- 后端：Python 3.10+
- Telegram Bot API：python-telegram-bot 库
- 数据库：SQLite/MySQL
- 支付接口：第三方支付 API
- 部署：Docker 容器化

### 6.2 系统架构
- 模块化设计，易于维护和扩展
- 采用异步处理，提高并发性能
- 数据库设计合理，确保数据一致性
- 完善的错误处理和日志记录

### 6.3 数据存储
- 用户信息：TG ID、用户名、余额、累计充值、累计申请
- 订单信息：订单号、用户 ID、金额、支付方式、状态、创建时间
- 卡密信息：卡密、面值、状态、创建时间、使用时间
- API 申请记录：用户 ID、手机号、api_id、api_hash、申请时间
