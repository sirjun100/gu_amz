# USB 客户端规划（EasyClick iOS USB · 亚马逊任务）

本文档只做**架构与流程规划**，不包含具体脚本实现。实现代码放在 `src/js` 下，与现有 `main.js` 入口衔接。

---

## 1. 项目定位

- **运行环境**：EasyClick **iOS USB 版**，JavaScript 脚本。
- **对接后台**：贵司 `amz` 运维服务（领取任务、心跳、随机关键词、上报日志与截图、`AMZ_REPORT` 结案协议等）。
- **业务目标**：在可控节奏下完成三类**广告点击类**任务（搜索点击 / 相关产品点击 / 同类产品点击），并合规上报执行结果与环境信息。

## 3. 规划中的 `src/js` 结构（建议）

EasyClick 支持 **中文标识符**（文件名、函数名、变量名）。为便于和 PRD / 测试用例对照分析，**`steps/` 与 `flows/` 建议全部采用中文命名**；`api`、`core` 仍可用英文短名，避免与官方 `http.js` 等示例混读时产生双重心智负担（也可全部中文化，按团队习惯二选一）。

```
src/js/
├── main.js                          # 入口：自动化服务、卡密、拉任务、分发流程、统一上报
├── config.js                        # 常量：API Base、device_id、超时、站点域名、包名等
│
├── api/
│   ├── amzClient.js                 # 心跳、领取任务、上报 report（与后端路径一致，英文便于搜接口）
│   └── 随机关键词.js                 # GET /client/random-keywords（若单独拆文件，可用中文文件名）
│
├── core/
│   ├── logCollector.js              # 日志缓冲 → log_lines；末尾追加 AMZ_REPORT（可改为 日志收集器.js）
│   ├── report.js                    # multipart 组装
│   └── session.js                   # 当前 taskId / params / task_type
│
├── flows/                           # 三条业务主流程：只做「步骤编排 + 超时/异常策略」，具体点击在 steps
│   ├── 关键词广告点击流程.js          # task_type: search_click
│   ├── 相关商品广告点击流程.js        # task_type: related_click
│   └── 同行同类广告点击流程.js        # task_type: similar_click
│
├── steps/                           # 可复用原子步骤：文件名 ≈ 业务动作，与第 5 节步骤表可逐条映射
│   ├── 打开AMG.js                  # 主要函数示例：打开AMG()、确保AMG在前台() 等
│   ├── 按顺序选择环境.js             # 按顺序选择环境()；入参可为环境序号或配置表
│   ├── 打开Chrome浏览器.js
│   ├── 打开亚马逊首页.js             # 打开亚马逊首页(站点配置)
│   ├── 检查账号是否已登录.js         # 检查账号是否已登录() → 未登录抛错或返回 false，由 flow 决定上报
│   ├── 首页随机浏览.js               # 首页随机浏览(分钟最小值, 分钟最大值)
│   ├── 关键词库随机搜索浏览或加购.js  # 关键词库随机搜索浏览或加购(分钟最小, 分钟最大)；内部拉随机词或传词列表
│   ├── 任务关键词搜索竞品进详情.js    # 任务关键词搜索竞品进详情(关键词, 停留分钟区间)；related/similar 共用
│   ├── 搜索并点击目标任务广告.js      # 依据 params 找目标商品 + 点广告 + 详情/店内浏览；仅 search_click
│   ├── 详情页相关商品广告点击.js      # Related 区块：翻页、筛品牌、随机点 2～3 个、每段浏览；related_click
│   └── 详情页同类推荐广告点击.js      # You might also like 等区块：逻辑同上；similar_click
│
└── utils/
    ├── 随机工具.js                   # 区间随机分钟、从数组随机取 n 个不重复等：随机区间()、随机抽取()
    └── 人类停顿.js                   # 人类化停顿()（可选）
```

### 3.1 `steps/` 与函数的约定（中文名）

| 文件名 | 建议主函数（示例） | 备注 |
|--------|-------------------|------|
| `打开AMG.js` | `打开AMG()` | 启动 / 切前台 |
| `按顺序选择环境.js` | `按顺序选择环境(配置)` | 与 AMG UI 步骤绑定 |
| `打开Chrome浏览器.js` | `打开Chrome浏览器()` | |
| `打开亚马逊首页.js` | `打开亚马逊首页()` | |
| `检查账号是否已登录.js` | `检查账号是否已登录()` | 见 Sign in 等特征 |
| `首页随机浏览.js` | `首页随机浏览(最小分钟, 最大分钟)` | |
| `关键词库随机搜索浏览或加购.js` | `关键词库随机搜索浏览或加购(最小分钟, 最大分钟)` | |
| `任务关键词搜索竞品进详情.js` | `任务关键词搜索竞品进详情(关键词, 最小分钟, 最大分钟)` | |
| `搜索并点击目标任务广告.js` | `搜索并点击目标任务广告(任务参数)` | 读 `params` |
| `详情页相关商品广告点击.js` | `详情页相关商品广告点击(品牌或ASIN条件, 点击个数区间, 浏览分钟区间)` | |
| `详情页同类推荐广告点击.js` | `详情页同类推荐广告点击(...)` | 参数结构可与上类似 |

内部可有**私有**辅助函数，同样建议中文命名，例如：`查找文本是否出现("Sign in")`、`在屏幕内缓慢滑动浏览()`。

### 3.2 `flows/` 编排与 `steps/` 的调用关系（示意）

- **`关键词广告点击流程.js`**：依次调用 `打开AMG` → `按顺序选择环境` → `打开Chrome浏览器` → `打开亚马逊首页` → `检查账号是否已登录` → `首页随机浏览` → `关键词库随机搜索浏览或加购` → **`搜索并点击目标任务广告`**。
- **`相关商品广告点击流程.js`**：前 7 步同上 → **`任务关键词搜索竞品进详情`** → **`详情页相关商品广告点击`**。
- **`同行同类广告点击流程.js`**：前 8 步同 related → **`详情页同类推荐广告点击`**。

`main.js` 中根据 `task.type` / `task_type` 映射到上述三个流程文件之一（可用 `if/else` 或中文函数名 `运行关键词广告点击流程()` 导出）。

### 3.3 其它说明

- `ios-jslibs`：仍为 **EasyClick 官方 API** 来源；`steps` 内只调官方能力与自封装的 `api/core/utils`。
- **文件编码**：建议 UTF-8，避免 IDE 与真机打包乱码。
- **中文文件名**：在部分命令行/旧工具下可能显示异常，团队开发时统一用 EasyClick IDE 管理工程即可。

## 4. 与服务端的约定（实现时必须遵守）

- 任务参数来自接口返回的 **`task.params`**（对象），例如点击类包含 `keyword`、`product_title` 等（以实际接口为准）。
- 结案时在日志数组**末尾**追加一行：  
  `AMZ_REPORT {"status":"success|failed","environment":"...","finished_at":"ISO8601","error":"..."}`  
  详见仓库根目录 **`客户端上报日志约定.md`** 与 **`说明.txt`** 中的引用。
- **未登录**：检测到 Sign in 等，应 **failed** 或 **success=false** 并带清晰 `error`，避免无意义继续跑广告点击。

## 5. 业务流程规划

以下步骤为**逻辑顺序**；真实脚本中需在每步记录日志、处理超时、截图（失败时尤其需要）。

### 5.1 关键词广告点击（`search_click`）

| 步骤 | 说明 |
|------|------|
| 1 | 打开 **AMG**（启动方式、包名、是否已安装以真机与文档为准） |
| 2 | **按顺序选择环境**（与 AMG 内 UI 约定一致，需固定选择顺序或通过配置映射） |
| 3 | 打开 **Chrome** 浏览器 |
| 4 | 打开 **Amazon 首页**（区域域名与语言以 `config` 为准） |
| 5 | **检查账号**：若页面出现 **Sign in** 等未登录特征 → **立即上报失败**，结束本任务 |
| 6 | **首页随机浏览** 2–4 分钟（随机落在该区间内） |
| 7 | **随机关键词**：来自「关键词数据表 1」— 实现上与 **服务端随机关键词接口** 或 **本地表** 二选一，需与产品确认；在亚马逊内 **搜索**，后续 **浏览或加购**，总时长 **2–5 分钟**（随机） |
| 8 | **找到目标商品**：依据任务 `params`（如 `product_title` / ASIN 等）与页面匹配；选择 **广告位** 进入目标商品；**店内/详情浏览** 2–3 分钟（随机） |

### 5.2 相关商品广告点击（`related_click`）

| 步骤 | 说明 |
|------|------|
| 1 | 打开 AMG |
| 2 | 按顺序选择环境 |
| 3 | 打开 Chrome |
| 4 | 打开 Amazon 首页 |
| 5 | 账号检查；未登录则上报结束 |
| 6 | 首页随机浏览 2–4 分钟 |
| 7 | **随机选关键词库 1** 的词 → 搜索 → 浏览或加购 → **2–5 分钟** |
| 8 | **根据任务**：使用任务下发的 **关键词**（`params.keyword`）搜索、浏览，找到 **竞争对手**，进入 **竞品详情页**，停留 **1–3 分钟** |
| 9 | 在详情页定位 **目标 ASIN 对应品牌** 的 **Related products（相关推荐/广告区块）**；**允许翻页**；随机点击 **2–3 个** 属于目标品牌的广告入口；每个进入后浏览 **1–3 分钟**（可每次随机） |

### 5.3 同行 / 同类商品广告点击（`similar_click`）

| 步骤 | 说明 |
|------|------|
| 1 | 打开 AMG |
| 2 | 按顺序选择环境 |
| 3 | 打开 Chrome |
| 4 | 打开 Amazon 首页 |
| 5 | 账号检查；未登录则上报结束 |
| 6 | 首页随机浏览 2–4 分钟 |
| 7 | 随机关键词库 1 → 搜索、浏览或加购 → **2–5 分钟** |
| 8 | 使用任务关键词搜索、浏览，找到竞争对手，进详情页 → **1–3 分钟** |
| 9 | 在详情页找到目标 ASIN 品牌在 **「You might also like / sponsored」** 类区块（文案以实际亚马逊 App/Web 为准）；可翻页；随机点 **2–3 个** 目标品牌广告；每段 **1–3 分钟** |

## 6. 三条流程序列对照（抽象相同与差异）

| 阶段 | search_click | related_click | similar_click |
|------|--------------|---------------|----------------|
| AMG + 环境 + Chrome + 首页 + 登录检测 + 首页漫游 + 关键词库 1 漫游 | 相同 | 相同 | 相同 |
| 任务关键词 → 找竞品 → 详情 | **不强调竞品链路**（直接找目标+广告） | **需要** | **需要** |
| 广告区块 | **目标商品广告点击** | **Related products** 目标品牌多个 | **You might also like / sponsored** 目标品牌多个 |

实现时建议：**共用 `steps/` 下中文模块**，仅在 `flows/` 三个中文流程文件里拼装第 7～9 步差异，减少重复。

## 7. 待产品与实现确认项（写代码前建议定稿）

1. **AMG** 的包名、首次启动权限、环境列表与会话编号是否与「顺序选择」一一对应（是否需配置表）。
2. **Chrome** 在 iOS 上是否固定为 Google Chrome；Amazon 使用 **m.amazon.com** 还是地区站、是否深链。
3. **「关键词数据表 1」**：是否等同于后端 **`/client/random-keywords`**；每次随机几条；是否与任务 `keyword` 区分使用场景。
4. **「目标商品 / 目标 ASIN / 品牌」** 在 UI 上识别规则：标题包含、ASIN 区块、Buy Box、图像匹配等（易触发维护成本与风控）。
5. **广告区块文案** 会随亚马逊改版变化；需约定 **多语言/多布局兜底**（OCR / 相对区域 / 关键词附近滑动）。
6. **加购** 是否必须执行：文案为「浏览或者加购」，需定义概率或 A/B。

## 8. 迭代顺序建议

1. 打通 **拉任务 → 日志 → AMZ_REPORT → report**（可用假任务 `params` 手动测）。
2. 在 `steps/` 中实现公共：**打开AMG**、**按顺序选择环境**、**打开Chrome浏览器**、**打开亚马逊首页**、**检查账号是否已登录**、**首页随机浏览**（可先 stub）。
3. 实现 **`关键词广告点击流程.js`** 全链路（最简单）。
4. 补齐 **`任务关键词搜索竞品进详情.js`**，再实现 **`详情页相关商品广告点击.js`** 与 **`详情页同类推荐广告点击.js`**，分别接入 **相关商品** / **同行同类** 两条流程。
